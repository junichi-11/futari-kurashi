# Production SOP

1. Phaseの目的と非目的を定義する。
2. CodexがDraft PRのブランチで実装する。
3. Product LibraryとMOSのValidationを実行する。
4. Preview / Production相当、PC / Mobile、Console、リンクを確認する。
5. 人間が画像、文章、価格、在庫、送料を目視する。
6. 指摘を修正し、必要なMOSを更新する。
7. GitHub Checks完了後にReady for reviewを判断する。
8. 承認された変更だけをmainへマージする。

参照元: `README.md`、`docs/release-checklist.md`、Phase 1〜8。最終レビュー: 2026-07-16。
