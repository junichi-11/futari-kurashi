# MARGIN 編集ガイドライン

MARGINは「余白から考える、ふたり暮らしの選択」を編集方針とします。商品を並べる前に、ふたりが過ごす場面、住まいの制約、手入れや使い続けやすさを整理します。

## 記事の原則

- 読者は新婚・同棲を始める、または見直すふたりを想定する。
- 「絶対」「必ず」など根拠のない断定や、不安を過度にあおる表現を避ける。
- 寸法や価格など変化する情報は出典・確認日を記録し、目安であることを明示する。
- 商品の長所だけでなく、向かない暮らしや注意点も同じ重さで扱う。
- メーカー文言の転載は避け、一次情報を確認したうえで編集部の言葉にする。
- 他者の著作物、画像、商標を尊重し、利用条件とクレジットを確認する。

## 記事構成

1. 読者の悩みと、記事で分かることを冒頭で示す。
2. 商品名より先に、選定軸と判断手順を説明する。
3. 数値は単位を付け、必要に応じて幅や条件を示す。
4. 購入前の確認事項をチェックリストにまとめる。
5. 広告・アフィリエイトの有無を記事内で明示する。

## 商品掲載

- `data/products.json` のValidationを通過した `publishable: true` の商品だけを公開記事に表示する。
- 商品URL、仕様、価格表現、在庫、画像利用条件を公開前に確認する。
- 報酬の有無にかかわらず、編集上の選定理由を説明できる商品だけを扱う。
- 提供品・広告・アフィリエイトは読者が記事本文に入る前、またはリンク付近で明瞭に表示する。

## データと更新

- 記事メタデータは `data/articles.json` を正とし、HTMLのtitle、description、canonical、構造化データと一致させる。
- 商品確認日はSource Registryの `last_verified` にISO 8601形式で記録する。
- 内容を変更した場合は記事とデータの `updatedAt`、必要に応じて `sitemap.xml` の `lastmod` を更新する。
